# 17 — Documentation Update

## Goal

Update documentation across all repos to reflect the new architecture.
Each repo's `CLAUDE.md` and `README.md` accurately describes its
scope, dependencies, and place in the OpenCDD ecosystem.

## Background

After the architecture migration, four repos have interrelated
documentation needs:

- `opencdd/cdd-data` — scope shrinks (no browser/editor); gains
  submodule + release pipeline.
- `opencdd/cdd-models-ts` — new repo; needs full package docs.
- `opencdd/opencdd.github.io` (this repo) — new Astro browser; needs
  CLAUDE.md describing the architecture, build, deploy.
- `opencdd/editor` — (future, per TODO.astro/16) new repo; needs
  full editor docs.

Cross-references between repos must be accurate so a developer
landing in any one repo understands how it fits.
